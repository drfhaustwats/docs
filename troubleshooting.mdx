# Troubleshooting Guide

This guide provides specific code locations, debugging procedures, and diagnostic steps for common issues in the Watts v3 dual-MCU system. Each section points directly to relevant source files and functions.

## AT Communication Issues

### UART Connection Problems

**Symptoms:**
- No response to AT commands
- Garbled responses
- Command timeouts

**Check These Code Locations:**

1. **ESP32-C5 ATHost Configuration** (`esp32-c5-main/ATHost.cpp`):
   ```cpp
   // Line ~45: UART initialization
   void ATHost_init() {
       uart_config_t uart_config = {
           .baud_rate = 5000000,           // Verify baud rate match
           .flow_ctrl = UART_HW_FLOWCTRL_CTS_RTS,  // Check flow control
       };
   }
   ```

2. **ESP32-H2 CAT Parser Setup** (`esp32-h2-at/main/EspAt_firmware.c`):
   ```cpp
   // Line ~82: UART configuration must match C5
   uart_config_t uart_config = {
       .baud_rate = 5000000,               // Must match ATHost
       .flow_ctrl = UART_HW_FLOWCTRL_CTS_RTS,
       .rx_flow_ctrl_thresh = 122,         // Check threshold
   };
   ```

**Debugging Steps:**
```bash
# Check H2 UART pins in hardware.mdx
GPIO 4  -> UART TX (to C5 RX)
GPIO 5  -> UART RX (from C5 TX) 
GPIO 6  -> UART RTS
GPIO 7  -> UART CTS

# Test basic connectivity
echo "AT" > /dev/ttyUSB1  # Should return "OK"
```

## Hardware Configuration Issues

### GPIO Pin Conflicts

**Symptoms:**
- Signals not reaching expected pins
- Multiple functions on same GPIO
- Incorrect voltage levels

**Check These Pin Assignments:**

1. **ESP32-C5 Pin Configuration** (see `hardware.mdx`):
   ```
   GPIO 4,5,6,7   -> UART to H2
   GPIO 10,26,25  -> ParlIO to H2 SPI
   GPIO 27,28     -> RMT LED outputs (channels 0,1)
   ```

2. **ESP32-H2 Pin Configuration** (see `hardware.mdx`):
   ```
   GPIO 4,5,6,7   -> UART from C5
   GPIO 10,11,25  -> SPI slave from C5 ParlIO
   GPIO 27,26     -> RMT LED outputs (channels 2,3)
   ```

**Verify Pin Configuration:**
```cpp
// In both MCU initialization
gpio_config_t io_conf = {
    .pin_bit_mask = (1ULL << gpio_num),
    .mode = GPIO_MODE_OUTPUT,           // Check mode
    .pull_up_en = GPIO_PULLUP_DISABLE,  // Check pull settings
    .pull_down_en = GPIO_PULLDOWN_DISABLE,
    .intr_type = GPIO_INTR_DISABLE
};
```
